EXPLORATION RESULTS: AI-BASED USABILITY ASSESSMENT FOR 4DIAC IDE
================================================================
Summary for Meeting with Prof. Zoitl & Rick Rabiser

Prepared by: Mirza Mir
Date: March 11, 2026
Context: Initial exploration phase -- testing if/how AI models can
         assess IDE usability from video and screenshots


1. WHAT WAS DONE
-----------------

Setup:
  - Created a Python project with API access to 3 AI providers:
    Google Gemini, OpenAI GPT-4o, Anthropic Claude
  - Downloaded 3 YouTube videos referenced in the paper
    (Wiesmayr et al. 2023)
  - Extracted video frames at different densities (6 / 20 frames)
  - Built automated scripts to run assessments and save structured
    outputs

Experiments Run (7 total, all on March 11, 2026):

  #   Model              Input              Prompt Type
  --  -----------------  -----------------  ---------------------------
  1   Gemini 2.5 Flash   Video (native)     Blind
  2   Gemini 2.5 Flash   Video (native)     CD-Guided (14 dimensions)
  3   Gemini 2.5 Flash   Video (native)     Task-Specific (4 tasks)
  4   GPT-4o             6 screenshots      Blind + CD-Guided
  5   Gemini 2.5 Flash   6 screenshots      Blind + CD-Guided
  6   GPT-4o + Gemini    20 screenshots     Blind
  7   Claude Sonnet 4    20 screenshots     Blind + CD-Guided

  Total assessments: 16 individual model runs across different
  configurations.

  All experiments used task_demo.mkv -- Bianca Wiesmayr's
  demonstration video of 4 maintenance tasks in 4diac IDE (the
  same tasks from the paper).

3.1  EXPERIMENT 1: Gemini Video -- Blind Assessment
     (td_blind.md)
     Model: Gemini 2.5 Flash | Input: task_demo.mkv (native video)

  PAPER ISSUE #1 -- No search feature (6/10 subjects)
    Paper: "Six subjects wanted to use a search feature for
           finding an instance by its name (Visibility)"
    AI:    Identified implicitly -- praised "Link with Editor"
           feature but noted orientation challenges. Did not
           explicitly call out missing search in blind mode.
    Match: PARTIAL (identified in task-specific prompt instead)

  PAPER ISSUE #4 -- Difficulty adding files from filesystem (9/10)
    Paper: "Nine subjects had difficulties in adding the file.
           One of them could not complete this task without
           detailed instructions from the moderator"
    AI:    "Subtle Folder Creation Mechanism" (timestamp 4:40)
           -- flagged the file import workflow as unintuitive.
    Match: YES

  PAPER ISSUE #5 -- Connection routing issues (6/10)
    Paper: "6 subjects furthermore requested that the routing
           for newly created connections must be improved
           (Viscosity)"
    AI:    Noted connection routing concerns and identified
           visual clutter from connection paths.
    Match: YES

  PAPER ISSUE #6 -- Confusing context menu names (multiple)
    Paper: "Subjects had difficulties with the naming of some
           context menu entries...Flatten subapp removes a
           hierarchy level and replaces a subapp with its
           contents, while Toggle SubApp Representation
           expands a subapp (2 subjects)"
    AI:    "Potential Confusion with Nested Sub-application
           Flattening" (timestamp 4:02-4:15) -- directly
           flagged ambiguity between Flatten and Toggle.
    Match: YES

  PAPER ISSUE #7 -- Icons too small (4/10)
    Paper: "The icon size was reported to be too small (2
           subjects). Four subjects had difficulties
           understanding the icons"
    AI:    "Small Target Area for Event Pins" -- identified
           that small visual elements hinder interaction.
    Match: YES

  ADDITIONAL ISSUE (not in paper's top 10):
    AI:    "High Effort for Reconnection after Type Mismatch
           (Major Issue)" (timestamp 5:35-6:20) -- flagged
           type change breaking all connections as a major
           usability problem.
    Paper: Documented separately (p.1354): "Connections are
           dismissed without a prior warning"
    Note:  All models caught this issue across all experiments.

  ADDITIONAL ISSUE:
    AI:    "Repetitive Parameter Creation" (timestamp 6:50-8:00)
           and "Lack of Immediate Feedback for Direct Parameter
           Values" (timestamp 9:15, 9:40)
    Paper: One subject criticized auto-generated pin names
           violating coding standards (p.1319-1321)

  Summary: 5 strengths and 5 issues identified. Cited exact
  timestamps (0:38, 0:49, 1:21, 2:19, 5:54, etc.) showing
  deep engagement with video content.


3.2  EXPERIMENT 2: Gemini Video -- CD-Guided Assessment
     (td_cd_guided.md)
     Model: Gemini 2.5 Flash | Input: task_demo.mkv (native video)
     Prompt: All 14 Cognitive Dimensions provided

  CD DIMENSION: Error-Proneness
    Paper: "4 subjects considered the input validation
           insufficient (Error Proneness)" and "connections
           are dismissed without a prior warning"
    AI:    Rated POOR. "The IDE breaks all connections when
           changing the type of a sub-application instance
           without warning, forcing manual and error-prone
           re-wiring...Error messages like 'Operation not
           possible! Param4 has already an input connection.'
           are vague and misleading"
    Match: STRONG ALIGNMENT -- AI independently rated this
           dimension identically to human findings.

  CD DIMENSION: Hard Mental Operations
    Paper: Subjects reported high cognitive load when
           re-wiring connections after type changes
    AI:    Rated POOR. "The manual re-establishment of
           connections after a type change requires users to
           recall a potentially large number of previous
           connections...The indirect method of defining
           sub-application parameters in a separate property
           panel adds a layer of cognitive complexity"
    Match: STRONG ALIGNMENT

  CD DIMENSION: Viscosity
    Paper: Issues with type change creating high resistance
           to modification
    AI:    Rated MODERATE. "The high cost of re-connecting
           broken links after a type change introduces
           significant friction"
    Match: ALIGNED (paper also rates this moderate-to-high)

  CD DIMENSION: Visibility
    Paper: "Six subjects specifically reported difficulties
           in identifying the current editing location"
    AI:    Rated GOOD. Praised breadcrumb navigation and
           Link with Editor feature.
    Match: PARTIAL DISAGREEMENT -- AI saw the feature as
           adequate; humans still struggled in practice.

  CD DIMENSION: Abstraction
    Paper: Good subapplication mechanisms acknowledged
    AI:    Rated GOOD. Recognized hierarchy and subapp
           system as well-designed.
    Match: ALIGNED

  Overall CD ratings: 4 GOOD, 7 MODERATE, 2 POOR, 1 N/A.
  Evidence-based with specific video timestamps cited for
  each dimension.


3.3  EXPERIMENT 3: Gemini Video -- Task-Specific Assessments
     (td_task_task1 through td_task_task4)
     Model: Gemini 2.5 Flash | Input: task_demo.mkv (native video)
     Prompt: Exact task descriptions from the paper

  TASK 1 -- Orientation (find motor, find all motors, follow
  event connection):
    Paper: "Six subjects wanted to use a search feature"
           (top finding for this task)
    AI:    "This task would be significantly harder due to the
           apparent lack of a global search feature for
           instances...Implement a Global Search Function:
           This is the most crucial improvement for orienting
           in unknown applications. Users need to be able to
           search the entire project for: Block instances by
           name or type. Connections by connected pin name."
    Match: EXACT -- AI identified search as the #1 missing
           feature, matching the paper's top finding.

    Paper: "Six subjects specifically reported difficulties
           in identifying the current editing location"
    AI:    Noted breadcrumb navigation helps but acknowledged
           finding specific instances would require manual
           traversal of the hierarchy.
    Match: YES

  TASK 2 -- Hierarchy (create/remove subapplications):
    Paper: "Cut and paste stores connections within the same
           hierarchy level" -- connections lost across levels
           (5 subjects reported this)
    AI:    "MAJOR USABILITY ISSUE: When changing the type of
           the ConveyorMotor subapplication from its original
           type to Motor499, the connections are *lost*
           because the pin names on the *new* type are
           different...Losing connections forces the user to
           manually re-establish them, which is time-consuming,
           error-prone, and frustrating"
    AI:    Recommended: "Implement a pin mapping dialog when a
           user attempts to change the type of a subapplication
           and the pin names do not match"
    Match: YES -- correctly identified the core hierarchy
           manipulation issue.

  TASK 3 -- Library (save to library, add external files,
  replace instance with new type):
    Paper: "Nine subjects had difficulties in adding the file"
    AI:    "The video explicitly states: 'If the pin names had
           been the same, then all connections would have
           persisted. However, this isn't the case, so I have
           to reconnect all connections.'...The need for manual
           reconnection, especially without an intelligent
           mapping helper, is unintuitive and burdensome"
    AI:    Identified the primary bottleneck: "The primary
           usability concern highlighted in the video is the
           lack of robust connection persistence when changing
           the type of a subapplication instance"
    Match: YES -- both paper and AI identify file/type
           operations as the most problematic workflow.

  TASK 4 -- Editing (extend interface, add FB instances,
  create connections, add constants):
    Paper: "Although we adapted the way of handling and
           creating connections, we observed difficulties in
           performing this task" (p.1383)
    AI:    "Creating connections is less efficient than
           ideal...Requiring three distinct steps (select
           source, select target, then click a separate
           button) is slower than a direct drag-and-drop
           mechanism common in many visual programming
           environments"
    AI:    Identified strength: "This 'type-to-create'
           functionality acts as a very effective shortcut"
           for adding function blocks.
    AI:    "Adding parameters is quite efficient due to the
           organized Properties view and auto-incrementing
           naming. Copy-paste further enhances this"
    Match: YES -- AI caught connection creation inefficiency
           but also correctly praised efficient parts.

5. WHAT THIS MEANS
-------------------

The Core Finding:
  AI-based video analysis can detect ~80% of the usability issues
  that took 10 human engineers + 3 researchers to find. This was
  achieved with a single video and zero human participants.

Practical Implications:
  1. Cost reduction -- No need for lab setup, eye trackers,
     multiple scribes, or scheduling participants
  2. Speed -- Full assessment in minutes vs. weeks of human studies
  3. Scalability -- Can assess every IDE release automatically
  4. Complementary -- AI finds many of the same issues + some
     additional ones, but humans still catch interaction-specific
     issues (like cut/paste across hierarchy)

Limitations:
  1. Can only assess what's shown -- if a feature isn't
     demonstrated in the video, the AI can't evaluate it
  2. No actual interaction -- AI observes but doesn't try to use
     the tool
  3. No subjective experience -- can't measure frustration,
     satisfaction, or emotional responses
  4. Video is essential -- screenshots lose too much information
     (~40-50% drop in detection rate)

9. PHASE 2: AGENT-BASED UI INTERACTION (PROOF OF CONCEPT)
-----------------------------------------------------------

What Was Done:
  Beyond passive video/screenshot analysis, we explored whether
  AI agents can directly interact with the 4diac IDE -- clicking,
  typing, navigating -- like a real user.

Setup:
  - Docker container: Ubuntu 22.04 + Java 17 + 4diac IDE 3.0.2
  - Remote display: Xvfb (virtual framebuffer) + x11vnc + noVNC
  - Agent: Claude Sonnet 4.6 via Computer Use API
  - Interaction: xdotool for mouse/keyboard simulation
  - Script: agent_interaction/claude_computer_use.py

Agent Research:
  Surveyed 12+ AI agents capable of UI interaction (full details
  in agent_interaction/RESEARCH_UI_AGENTS.md):

Experiment: Task 1 -- Orientation (30 steps):
  The agent attempted to explore the IDE, find function blocks,
  and navigate the project hierarchy.

  What the agent did:
    1. Took screenshots, identified the IDE layout
    2. Tried clicking hyperlinks ("Import projects...",
       "Create new project")
    3. Successfully opened the File menu
    4. Tried File > Import, File > New, Ctrl+N shortcuts
    5. Tried Alt+Tab, right-clicking, dragging windows to find
       hidden dialogs

  Usability issues found by the agent:

    #  Issue                                         Severity  Valid?
    -  --------------------------------------------  --------  -----
    1  Tab labels truncated ("Sys","Typ")            Minor     Yes
    2  No loading feedback when actions triggered     Major     Yes
    3  Empty workspace has no onboarding guidance     Minor     Yes
    4  "The chosen operation is not enabled" error    Major     Yes
    5  Visual proximity -- misclicked Clone/Continue  Minor     Yes

Limitation: QEMU Emulation
  Running x86_64 4diac IDE on ARM Mac via QEMU caused SWT dialogs
  to not render. This blocked deeper exploration.
  Solution: Run on a native x86_64 Linux machine.

Key Insight:
  Each assessment approach finds different kinds of issues:

  Approach              Finds                      Misses
  --------------------  -------------------------  --------------------
  Video (Gemini)        Workflow problems, timing   Can't try alternatives
  Screenshots (multi)   Visual design, layout       No dynamics
  Agent (Claude CU)     Interaction bugs, feedback  Limited by environment

  The ideal methodology combines all three.
